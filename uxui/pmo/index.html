Challenge

The challenge was to design a To-do, scheduling application for parents with young kids. Reserach found a plethora of Apps available. Many which are aimed at self-development or gaming and educational apps, that help keep young children distracted whilst parents are busy. There was very few Apps that incorpated both a To-do list or memo along with scheduling daily tasks, activities, appointments and extracurricular activities. The main focus of this design was to follow the KISS principle (Keep it Simple Stupid), but not too simple.

Ideation

Every design begins with sketches and compiling research notes often scribbled down for speed and creative flow. This is the first evolution of design incorpating the knowledge from research along with the projects criteria and requirements. The main feature was to be able to have this application sync with mobile device calendars and potentially utilise voice to text technlogy so events and memos could be schedulede and saved hands free.

User flow

Integral to understanding the user journey, User flows are created to help guide the application build ensuring there is a natural flow and ease of use for the user. Concise and clear accessibility is of the utmost importance. This also helps decipher how key features will be applied to the application.

Wire frames

Wireframes both hand drawn or digital versions allow the designers to test and get a visual feel of how the App will work. This is where backbones of the application are designed along with ironing out any potential design flaws.

Style guide

During the research and ideation process, it is important to define a style guide. The Style guide is a reference tool for ensuring the project has a consistant visual appeal and meets the criteria of each project. This process can change the whole look of the original wireframes design and feel.

Prototypes

Prototypes generated in Figma allow for peer and client reviews. It is an essential part of the process for testing and refining designs. This project's aim was to have a simple UI where the application could be navigated without leaving the dashboard and having all features accessible with the fewest clicks possible.

Demo

Video demonstration's exported from the prototype exhbit how ther user will interact with the application. An important part of user testing to ensure the application functions as expected and deisgn refinements can be made.
